# Rules: Style

## Apply When
- Code changes.

## Do
- Follow existing repo conventions and linters.
- Prefer clarity over cleverness.
- Add comments only for non-obvious logic.

## Don't
- Don't reformat unrelated code.
- Don't rename or reorganize without a clear need.
