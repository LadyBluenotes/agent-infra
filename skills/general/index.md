# Code

## Apply When
- Implementing features, fixes, refactors, automation, or tooling.

## Do
- Use the smallest workflow module that fits:
  - Implement/change: @skills/general/implement.md
  - JavaScript deep dives: @skills/javascript/index.md
  - SolidJS deep dives: @skills/frameworks/solid/index.md
  - TypeScript deep dives: @skills/typescript/index.md
  - Performance work: @skills/performance/index.md
  - Validate (tests/build): @skills/general/validate.md
  - Communicate results: @skills/general/communicate.md

## Playbooks
- Implement: @skills/general/implement.md (making or refactoring code changes)
- Validate: @skills/general/validate.md (tests/build/typecheck after changes)
- Communicate: @skills/general/communicate.md (reporting what changed and how to verify)
- Simplify: @skills/general/simplify.md (readability-only refactors without behavior changes)
- Performance: @skills/performance/index.md (targeted optimization with measurement)
- Accessibility: @skills/general/accessibility.md (UI/UX accessibility checks)
- JavaScript: @skills/javascript/index.md (JS runtime patterns and module usage)
- TypeScript: @skills/typescript/index.md (type modeling, config, TS-specific fixes)
- SolidJS: @skills/frameworks/solid/index.md (Solid-specific patterns and primitives)

## Don't
- Don't broaden the scope (refactors, dependency bumps) unless requested.

## Output
- Include touched paths and verification commands.
