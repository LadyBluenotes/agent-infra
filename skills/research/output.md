# Output

## Setup
Use this when presenting research findings.

## Core Patterns

### Answer first
Lead with a concise recommendation or result.

### Tradeoffs
Summarize the key pros/cons and when an alternative is better.

### Unknowns
Call out any gaps that could change the recommendation.

### Next steps
Suggest validation steps or experiments to de-risk the decision.

## Common Mistakes

### Burying the answer
Wrong
```text
"Here is a long comparison..." (no recommendation).
```
Correct
```text
"Recommendation: use X for reason Y, with caveat Z."
```
Explanation: The reader should not have to infer the conclusion.

### Ignoring uncertainties
Wrong
```text
"This will work for all cases" (no caveats).
```
Correct
```text
"Unknowns: performance on large datasets; validate with a pilot."
```
Explanation: Uncertainty should be explicit so decisions are informed.
