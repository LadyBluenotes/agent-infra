# Reference
## Apply When
- Documenting stable facts: APIs, flags, config, schemas.

## Do
- Be precise and complete.
- Use consistent, scannable structure.
- Define inputs/outputs, defaults, and edge cases.

## Don't
- Don't include recommendations or narrative.
- Don't include long tutorials.

## Output
- Prefer tables/lists.
- Include: name, purpose, type, default, constraints, examples (only if required).
