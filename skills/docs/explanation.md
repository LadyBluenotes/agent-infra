# Explanation

## Apply When
- Building understanding: concepts, rationale, architecture.

## Do
- Explain "why" and "how" using a clear mental model.
- Define terms and relationships.
- Call out tradeoffs and constraints.

## Don't
- Don't give step-by-step instructions.
- Don't present opinions as facts.

## Output
- Template:
  - Problem/Context
  - Model
  - Tradeoffs
  - Related links
