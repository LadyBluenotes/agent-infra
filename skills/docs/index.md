# Docs

## Apply When
- Writing or updating documentation.

## Do
- Pick exactly one Diataxis type.
- Apply the corresponding module:
  - Tutorial: @skills/docs/tutorial.md
  - How-to: @skills/docs/how-to.md
  - Reference: @skills/docs/reference.md
  - Explanation: @skills/docs/explanation.md

## Playbooks
- Tutorial: @skills/docs/tutorial.md (beginner, end-to-end learning path)
- How-to: @skills/docs/how-to.md (task-focused steps for a known goal)
- Reference: @skills/docs/reference.md (precise, exhaustive facts)
- Explanation: @skills/docs/explanation.md (conceptual understanding and rationale)

## Don't
- Don't mix Diataxis types in one doc.

## Output
- State the doc type at top.
- Provide final text, ready to commit.
