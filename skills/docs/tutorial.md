# Docs: Tutorial

## Apply When
- Teaching a beginner to succeed end-to-end.

## Do
- Start with prerequisites (tools, access, versions).
- Use a linear sequence of small steps.
- Show expected results after key steps.
- Keep "why" minimal; focus on "do this".

## Don't
- Don't branch into alternatives.
- Don't turn it into a reference list.

## Output
- Template:
  - Title
  - Goal
  - Prerequisites
  - Steps
  - Verification
  - Next steps
