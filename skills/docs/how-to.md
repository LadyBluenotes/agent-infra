# Docs: How-to

## Apply When
- Helping a user complete a specific task.

## Do
- Assume basic familiarity; skip onboarding.
- Organize by goal and constraints.
- Include commands/config snippets needed to complete the task.
- Include verification and common failure modes.

## Don't
- Don't teach fundamentals.
- Don't attempt to be exhaustive.

## Output
- Template:
  - Title
  - When to use this
  - Steps
  - Verify
  - Troubleshooting
